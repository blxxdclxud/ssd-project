Background - Same-Origin Policy & how CORS relaxes it

Browsers enforce the Same-Origin Policy (SOP): a script may freely read responses only from its own origin. An origin is the triplet scheme + host + port - all three must match exactly.

URL Same origin as http://frontend.local:8080? Reason
http://frontend.local:8080/other-page Yes scheme, host and port are identical
http://api.local:8081/api/data No different host (api.local) and port (8081)
http://evil.local:8082/ No different host and port
https://frontend.local:8080/ No different scheme (https vs http)

Without SOP any website you visit could silently read your emails, bank balance, or session tokens from other open tabs. CORS is a controlled relaxation: the server opts in by returning Access-Control-Allow-Origin in its response headers. If those headers are absent the browser blocks JavaScript from reading the response, even when the HTTP request itself already reached the server.

Simple vs non-simple requests. A plain GET or a POST with a basic content type is simple - the browser sends it directly and then checks the response headers. Any request with a non-trivial method or content type (e.g. Content-Type: application/json) is non-simple and triggers a preflight: the browser automatically sends an OPTIONS request first. Only if the server explicitly approves that preflight does the browser send the actual request. A missing CORS configuration therefore prevents the real non-simple request from ever leaving the browser.
